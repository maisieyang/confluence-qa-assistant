# 团队沟通规范

## 沟通工具

| 工具 | 用途 | 使用场景 |
|------|------|---------|
| **Slack** | 日常沟通 | 技术讨论、快速提问、非正式沟通 |
| **Zoom** | 视频会议 | 站会、1:1、技术评审 |
| **Confluence** | 文档 | 技术方案、会议纪要、操作手册 |
| **JIRA** | 项目管理 | 任务跟踪、Sprint 规划、Bug 管理 |
| **Email** | 正式沟通 | 跨部门通知、外部合作、合同相关 |

## Slack 使用规范

### 重要频道

| 频道 | 用途 |
|------|------|
| `#general` | 全公司公告 |
| `#engineering` | 工程团队通知和讨论 |
| `#tech-talks` | 技术分享报名和通知 |
| `#it-support` | IT 支持和问题反馈 |
| `#incidents` | 线上事故通报和处理 |
| `#alerts-critical` | 自动告警通知 |
| `#ai-platform` | AI 平台相关讨论 |
| `#random` | 非工作话题，自由交流 |

### Slack 礼仪
1. **@channel / @here** 仅用于紧急事项，请勿滥用
2. **线程回复**：使用 Thread 功能回复，保持频道整洁
3. **表情符号反馈**：用 👍 表示已收到，✅ 表示已完成
4. **工作时间**：非紧急消息请在工作时间（9:00-18:00 SGT）发送
5. **DM vs 公开频道**：技术问题尽量在公开频道讨论，方便他人搜索参考

## 会议规范

### 站会（Daily Standup）
- **时间**：每日 10:00 AM，15 分钟以内
- **格式**：每人 2 分钟，说三件事：
  - 昨天完成了什么
  - 今天计划做什么
  - 有什么阻碍
- **注意**：站会不是讨论问题的场合，有具体问题会后单独沟通

### 1:1 会议
- **频率**：与直属 Manager 每两周一次，30 分钟
- **内容**：职业发展、工作反馈、困难和挑战、个人目标

### Sprint 规划会
- **时间**：每两周一次，周一上午
- **时长**：1-2 小时
- **产出**：Sprint Backlog，每个任务有明确的 Assignee 和 Story Points

## 文档规范

### 什么时候必须写文档
- 新服务的架构设计（ADR）
- API 接口变更
- 运维操作手册
- 事故复盘报告
- 重要的技术决策

### 文档模板
所有文档模板存放在 Confluence `Templates` Space：
- 技术方案模板（RFC）
- 事故复盘模板（Post-Mortem）
- 会议纪要模板
- 新人入职 Checklist
